# CLIP On-Device Setup

One-time setup to enable cloud recognition. Without these steps the app
still runs — it falls back to a deterministic stub picker that always
returns the same 5 placeholder picks. With them, recognition is fully
on-device, $0 per call, no network, no vendor.

See `docs/RECOGNITION.md` for why this is the architecture.

---

## What you need to add

Two files, neither of which is in the git repo:

1. **`CloudView/Resources/MobileCLIP-S0.mlmodelc/`** — the Core ML image
   encoder (~26 MB). Apple ships this as part of their MobileCLIP
   release.
2. **`CloudView/Resources/LabelEmbeddings.json`** — pre-computed text
   embeddings for the kid-safe label allowlist (~300 KB). Generated by
   running the offline tool below.

Both go into a `Resources` group in the Xcode project (create it under
the `CloudView` target if it doesn't exist; mark both files for the
`CloudView` target so they're copied into the app bundle).

---

## Step 1: download MobileCLIP

```bash
git clone https://github.com/apple/ml-mobileclip.git /tmp/ml-mobileclip
# Apple ships pre-converted Core ML packages here:
ls /tmp/ml-mobileclip/checkpoints/   # confirm mobileclip_s0.mlpackage
```

If the Core ML packages aren't in the repo at the time you check, Apple
also publishes them on Hugging Face under `apple/MobileCLIP-S0`. Either
way, you want the **image encoder** half of MobileCLIP-S0 — the text
encoder runs offline only.

Drag the `.mlpackage` into Xcode under `CloudView/Resources/`. Xcode will
ask whether to compile it; say yes. The resulting `.mlmodelc` directory
is what `CLIPImageEncoder.swift` looks for.

Confirm it's added to the **CloudView** target's Copy Bundle Resources
build phase.

---

## Step 2: generate the label embeddings

```bash
# One-time: install the offline dependencies
pip install torch open_clip_torch

# Run the generator. ~30 seconds on any laptop.
python tools/generate_label_embeddings.py
```

This:
1. Reads the label allowlist from `backend/api/lib/allowlist.ts` (single
   source of truth — never duplicate the list).
2. Embeds 5 phrasings per label, averages them, L2-normalizes.
3. Writes `CloudView/Resources/LabelEmbeddings.json`.

Add the JSON file to the Xcode project under `CloudView/Resources/` and
confirm it's in the **CloudView** target's Copy Bundle Resources build
phase.

---

## Step 3: regenerate when the allowlist changes

Every time you add or remove labels from `KID_SAFE_LABELS` in
`backend/api/lib/allowlist.ts`, re-run:

```bash
python tools/generate_label_embeddings.py
```

…and re-commit the resulting `LabelEmbeddings.json`. The file is small
enough (~300 KB) that committing it is fine; we don't need it gitignored.

A future commit could automate this as a Git pre-commit hook or a CI
job, but for now it's manual to keep the dependency story clean (no
PyTorch in CI).

---

## Verifying

When the app launches with both files present, the Xcode console should
be silent on the CLIP front. If you see either of these warnings:

```
⚠️  MobileCLIP-S0.mlmodelc not found in bundle. CLIP recognition
   disabled; falling back to stub picks.

⚠️  LabelEmbeddings.json not found in bundle. CLIP recognition
   disabled; falling back to stub picks.
```

…then either file is missing or wasn't added to the Copy Bundle Resources
build phase. Check the project's `CloudView` target → Build Phases →
Copy Bundle Resources and confirm both appear there.
